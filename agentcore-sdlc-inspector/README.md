# AgentCore SDLC Inspector (POC)

Local inspector UI for observing in-flight AgentCore SDLC pipeline sessions. Lists active sessions from DynamoDB and provides buttons to check if Claude is running or stop a session.

## Quick Start

```bash
cd agentcore-sdlc-inspector
npm install
bash setup-env.sh          # fetches table name from CDK stack outputs → .env
npm run dev
```

Open http://localhost:5173 in your browser.

## Setup

### 1. Deploy the CDK stack (creates the DynamoDB table)

```bash
npx cdk deploy agent-assisted-sdlc-pipeline-assistant
```

### 2. Generate `.env`

```bash
bash setup-env.sh
# or with a custom stack name:
bash setup-env.sh my-custom-stack-name
```

This writes `.env` with `AWS_REGION` and `SESSIONS_TABLE_NAME` from the stack outputs.

### 3. Run

```bash
npm run dev
```

Starts the API server (port 8787) and the frontend dev server (port 5173).

## Environment Variables

Set in `.env` (auto-generated by `setup-env.sh`) or your shell:

| Variable | Default | Description |
|----------|---------|-------------|
| `AWS_REGION` | `us-west-2` | AWS region |
| `SESSIONS_TABLE_NAME` | — | DynamoDB table name (from CDK output) |
| `INSPECTOR_API_PORT` | `8787` | API server port |
| `INSPECTOR_WEB_PORT` | `5173` | Frontend dev server port |

## AWS Credentials

Uses the AWS SDK default credential chain (`~/.aws/credentials`, `AWS_PROFILE`, etc.). Your IAM principal needs:

- `dynamodb:Scan`, `dynamodb:Query` on the sessions table
- `bedrock-agentcore:InvokeAgentRuntimeCommand` on the runtime ARN (Check Claude running)
- `bedrock-agentcore:StopRuntimeSession` on the runtime ARN (Stop Session)

## Features

- **Time window filter** — 10 min, 1 hour, 1 day, or All
- **Auto-refresh** — every 15 seconds (pauses when tab is hidden)
- **Check Claude running** — probes the runtime microVM's `/proc` table, matches the specific `claude_session_uuid` to distinguish pipeline sessions from interactive ones
- **Stop Session** — stops the AgentCore runtime microVM (persistent storage survives)
- **Copy buttons** — for AgentCore Session ID and Claude Session UUID
- **Issue link** — title links to the issue URL (GitHub, Jira, etc.)
- **Per-invocation rows** — each pipeline invocation is a separate DynamoDB entry

## Architecture

- **API** (Express + TypeScript): `/api/health`, `/api/sessions?window=<minutes>`, `/api/check-claude`, `/api/stop-session`
- **Frontend** (React + Vite + TypeScript): Session cards with action buttons

## POC Posture

- **Local-only.** No deployment plumbing (no API Gateway, CloudFront, Cognito).
- **Scan cost.** The API uses DynamoDB `Scan` with a filter on `started_at`. For 1-2 concurrent users this is fine. If the table grows or many users run the inspector, add a GSI on `started_at` and switch to `Query`.
- **Simplified event stream parsing.** The API reads the AgentCore response and extracts JSON payloads via regex. Production should use `@smithy/eventstream-codec`.
- **No CI coverage.** The inspector tree is not part of the main repo's CI.

## DynamoDB Schema

| Field | Type | Description |
|-------|------|-------------|
| `session_id` | S (PK) | AgentCore runtime session ID |
| `invocation_number` | N (SK) | 1-based invocation counter |
| `runtime_arn` | S | Coding assistant runtime ARN |
| `assistant_type` | S | `claude-code`, `kiro`, `codex` |
| `repo_owner` | S | Repository owner |
| `repo_name` | S | Repository name |
| `issue_number` | N | Issue number |
| `issue_title` | S | Issue title (truncated to 200 chars) |
| `issue_url` | S | Link to the issue |
| `triggered_by` | S | GitHub username or "unknown" |
| `is_reinvocation` | BOOL | Whether this is a re-invocation |
| `claude_session_uuid` | S | UUID5 of the session ID |
| `started_at` | S | ISO 8601 timestamp |
| `ttl` | N | Auto-expire after 7 days |
